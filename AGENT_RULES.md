# AI 에이전트 개발 규칙

> 모든 AI 에이전트는 작업 시작 전에 이 파일을 읽고 따른다.
> 언어, 프레임워크, 프로젝트 종류와 무관하게 공통 적용된다.

---

## 1. 파일 수정 전 필수 절차

### 기존 파일을 수정하기 전에 반드시 먼저 선언한다

```
"XXX 파일의 YYY 부분을 수정합니다. 이유: ZZZ"
```

- 선언 없이 기존 파일을 수정하지 않는다
- 수정 범위가 **함수 1개를 초과**하면 사용자에게 확인을 받는다
- **파일 전체 재작성(Write)은 금지** — 반드시 부분 수정(Edit)만 사용한다

### 왜 중요한가

에이전트는 "지금 작업하는 부분"에 집중하면서 나머지 코드가 맥락에서 밀려난다.  
파일 전체를 재작성하면 기존 필드, 옵션, 로직이 조용히 삭제되는 사고가 반복된다.  
실제 사례: `/api/coupang/register` 핸들러 재작성 시 `delivery_company_code` 등 11개 필드가 통째로 삭제 → 배포 후에야 발견.

## 2. 리팩토링과 기능 추가를 동시에 하지 않는다

**금지 패턴:**
```
새 기능 추가 + 기존 파일 구조 변경을 하나의 작업으로 처리
```

**올바른 순서:**
```
1단계: 리팩토링만  → 커밋 → 동작 확인
2단계: 기능 추가만 → 커밋
```

## 3. 아키텍처 원칙

### 진입점 파일은 얇게 유지한다

`index.ts`, `main.py`, `app.js` 등 진입점 파일의 역할:
- **허용**: import, 라우트 등록, 미들웨어 연결
- **금지**: 비즈니스 로직, 직접적인 DB 쿼리, API 호출

**목표 크기: 100줄 이내**  
100줄을 초과하면 분리가 필요하다는 신호다. 작업 전에 먼저 제안한다.

### 도메인별 파일 분리

새 기능은 기존 파일 수정이 아닌 **새 파일 생성**을 우선으로 한다.

**Cloudflare Workers 예시 (현재 프로젝트):**
```
workers/src/
  index.ts          ← 라우팅만 (100줄 이내 목표)
  routes/
    coupang.ts      ← /api/coupang/* 핸들러
    nonoprice.ts    ← /api/nonoprice/* 핸들러
    costco.ts       ← /api/costco/* 핸들러
  coupang.ts        ← 쿠팡 비즈니스 로직 (기존 유지)
  nonoprice.ts      ← 노노프라이스 비즈니스 로직 (기존 유지)
  ...
```

**범용 백엔드 예시:**
```
src/
  index.ts / app.py   ← 앱 초기화 + 라우터 등록만
  routes/             ← 요청/응답 처리 (라우트별 파일)
  services/           ← 비즈니스 로직
  types/              ← 타입 정의
```

**단일 HTML 프론트엔드 (현재 프로젝트 pages/src/index.html):**
- 1개 파일이 1,900줄 이상 → 함수 추가 시 기존 함수 덮어쓰기 위험
- `<script>` 블록을 기능별로 명확히 주석 구분하고, 함수 추가는 **기존 블록 안에 append** 방식으로
- 전체 재작성 절대 금지

## 4. 커밋 규칙

### 커밋은 작고 자주

```
✅ "fix(coupang): add missing await on convertToCoupangProduct"
✅ "feat(costco-search): add /api/costco/search route"
❌ "여러 기능 수정 및 버그 수정"
```

### 코드 변경 → 즉시 커밋 → PR 업데이트 순서를 반드시 지킨다

변경 후 미커밋 상태로 다음 작업을 시작하지 않는다.

### Cloudflare Pages 배포 시 주의

커밋 메시지에 **한글이 포함되면 Pages API가 400 오류를 반환**한다.  
`wrangler pages deploy` 전 커밋 메시지는 반드시 영문(ASCII)으로 작성한다.

## 5. 배포 전 체크리스트

작업 완료 후 배포 전에 반드시 확인:

```
[ ] 기존에 동작하던 엔드포인트/기능이 그대로인가?
[ ] 수정한 파일에서 기존 필드/옵션/인자가 누락되지 않았는가?
[ ] async 함수 호출에 await가 있는가?
[ ] import에서 새로 쓰는 함수를 추가했는가?
[ ] 커밋 메시지가 영문인가? (Pages 배포 시)
```
